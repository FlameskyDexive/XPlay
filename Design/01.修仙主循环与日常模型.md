# 01. 修仙主循环与日常模型

## 1. 模块目标

- 建立“不玩SLG也能长期推进”的核心体验闭环。
- 支撑玩家在 `10-30分钟` 的碎片时间内完成关键成长动作。
- 将“放置收益、构筑决策、挑战反馈”三类行为整合为同一循环。

## 2. 核心玩法闭环

### 2.1 主闭环（纯修仙）

1. 挂机修炼/洞府产出获得修为与材料。
2. 调整功法、法宝、阵法与护道人，形成渡劫构筑。
3. 发起小层突破或大境跨越挑战（雷劫/天劫）。
4. 成功后解锁新副本、新养成位与新资源层。
5. 回到更高效率的挂机与挑战阶段。

### 2.2 补充闭环（可选SLG）

- 宗门协作提供效率增益、荣誉与外观。
- SLG产出仅作为“速度放大器”，不构成主线前置门槛。

### 2.3 非SLG替代循环

- 以个人周常、境界试炼、洞府巡修、遗迹词条收集承接中期目标。
- 不参与宗门时，玩家仍应持续获得“本周要冲什么、下周能解锁什么”的明确反馈。
- 替代循环补的是长期目标与基础效率，不补宗门荣誉、组织归属和展示性奖励。

## 3. 日常时间模型细化

## 3.1 轻度日常（10-15分钟）

- 收取离线收益（打坐/洞府/游历）。
- 处理1次小层突破或准备下一次渡劫材料。
- 完成基础日常任务（3-5个轻任务）。

## 3.2 标准日常（20-30分钟）

- 额外完成2-3个短副本（雷池秘境/境界试炼）。
- 调整功法流派与铭纹词条。
- 冲击天梯/心魔幻境1-2次。

## 3.3 深度游玩（45分钟+）

- 冲击高难雷劫（含心魔机制）。
- 深刷遗迹Roguelike并追求构筑上限。
- 参与宗门协作Boss（异步，不强制同屏在线）。

## 3.4 周节奏分配

- 标准周节奏中，修仙/PVE内容应占玩家有效操作时长的 `>=80%`。
- 宗门/SLG参与建议压缩为预约、派遣、收取战报、周结算四类轻行为。
- 不参与SLG的玩家应由个人榜、遗迹周目标、洞府升级计划承接相同时间预算内的成长反馈。

## 4. 状态机与关键流程

## 4.1 角色成长状态机

`Idle -> Cultivating -> BreakReady -> Tribulation -> Settle -> Idle`

## 4.2 关键判定点

- `BreakReady`：修为达标 + 材料达标 + 前置挑战完成。
- `Tribulation`：根据境界/层数生成雷劫参数与机制池。
- `Settle`：成功推进层数；失败执行返还与软保底累计。

## 5. 代码设计要点（ET）

## 5.1 建议包归属

- `cn.etetet.cultivation`：主循环编排、修炼与突破状态。
- `cn.etetet.tribulation`：雷劫实例、机制结算、失败补偿。
- `cn.etetet.adventure`：主线副本、试炼、遗迹与天梯。

## 5.2 建议组件（数据）

```csharp
public class CultivationComponent : Entity, IAwake
{
    public int RealmId;
    public int RealmStage;
    public long CultivationValue;
    public long LastTickTime;
    public int OfflineQueueHours;
    public int ContinuousBreakFailCount;
}
```

```csharp
public class DailyActionComponent : Entity, IAwake
{
    public int DailyTaskProgress;
    public int DailyBreakCount;
    public int WeeklyTrialCount;
    public long LastRefreshTime;
}
```

## 5.3 建议系统（逻辑）

- `CultivationComponentSystem`：修为累积、离线补算、修炼队列推进。
- `BreakthroughFlowSystem`：突破条件校验、状态流转、资源扣返。
- `DailyRefreshSystem`：日/周重置、活跃度结算、红点刷新。

## 5.4 异步安全要点

- 涉及 `await` 的突破与副本结算流程，必须使用 `EntityRef<T>` 重新获取实体。
- 不在 `Entity` 中写业务方法，复杂流程放到 `System` 或 `Helper`。

## 6. 配置与协议

## 6.1 必要配置表

- `RealmConfig`、`RealmBreakConfig`、`CaveConfig`。
- `DailyTaskConfig`、`WeeklyGoalConfig`。

## 6.2 核心协议

- `C2M_CultivateStart` / `M2C_CultivateSync`
- `C2M_RealmBreakTry` / `M2C_RealmBreakResult`
- `C2M_CaveCollect` / `M2C_CaveStatus`

## 7. 体验与监控指标

- 日均突破次数（按境界段分层）。
- 离线收益领取率与队列使用率。
- 48小时卡关占比与缓解后回流率。
- 纯修仙玩家30日留存与中期目标完成率。
- 非SLG玩家周目标完成率与30日目标完成率。
