# 05. PVE内容池与挑战玩法

## 1. 模块目标

- 提供纯修仙玩家长期可玩的挑战内容池。
- 让不同时长玩家都能找到“短平快/中强度/高挑战”目标。
- 与境界突破、材料产出、构筑成长形成联动。

## 2. 内容池分层设计

## 2.1 主线章节

- 功能：叙事推进 + 系统解锁 + 基础资源供给。
- 结构：普通关、精英关、章节首领关。
- 目标：每章至少对应一个成长里程碑（新境界/新系统）。

## 2.2 境界试炼

- 每个大境界一条独立Boss链。
- 作为跨境突破前置验证，强调构筑达标。
- 首通提供阶段性奖励，重复挑战偏向材料与熟练度收益。

## 2.3 雷池秘境

- 专产渡劫材料与抗雷法宝胚子。
- 设计多难度门槛，对应不同境界段。
- 每周限定高收益次数，防止资源通胀。

## 2.4 太古遗迹（Roguelike）

- 随机地图 + 词条分支 + 临时构筑。
- 鼓励尝试流派而非固定答案。
- 产出以铭纹、特殊词条、稀有材料为主。

## 2.5 无尽天梯

- 长线PVE排行，按个人战力分段匹配。
- 提供周结算目标与赛季荣誉。
- 不与SLG绑定，避免社交压力劝退。

## 2.6 心魔幻境

- 高风险高收益玩法，失败成本可控。
- 引入“心魔机制”检验道心与策略。
- 产出高价值铭纹与心魔抗性材料。

## 3. 节奏与奖励设计

- 日常：短副本（主线复刷/雷池）可10分钟完成。
- 周常：境界试炼与天梯冲层作为中期目标。
- 赛季：遗迹词条池与主题Boss轮换，提供新鲜度。

## 4. 代码设计要点（ET）

## 4.1 建议包归属

- `cn.etetet.adventure`：章节、试炼、遗迹、天梯、幻境。
- `cn.etetet.cultivation`：突破前置验证事件接入。

## 4.2 建议组件

```csharp
public class AdventureProgressComponent : Entity, IAwake
{
    public int MainChapterId;
    public int MainStageId;
    public Dictionary<int, int> TrialBossProgress;
    public int EndlessFloor;
}
```

```csharp
public class RogueLikeRunComponent : Entity, IAwake, IDestroy
{
    public int RelicMapSeed;
    public int CurrentNodeId;
    public List<int> TemporaryAffixIds;
    public int RiskScore;
}
```

## 4.3 建议系统

- `AdventureStageSystem`：关卡校验、掉落结算、首通奖励。
- `RealmTrialSystem`：境界Boss链流程与跨境前置标记。
- `RogueLikeRunSystem`：遗迹流程管理、词条抽取、结算。
- `EndlessTowerSystem`：爬塔匹配分组、周赛季结算。

## 4.4 重置与赛季化

- 天梯按周结算。
- 遗迹词条池按赛季更新（4-6周）。
- 仅重置赛季资源，不重置永久境界进度。

## 5. 配置与协议

## 5.1 配置表建议

- `AdventureStageConfig`
- `RealmTrialConfig`
- `ThunderPoolConfig`
- `RelicNodeConfig`
- `EndlessTowerConfig`

## 5.2 协议建议

- `C2M_AdventureEnter` / `M2C_AdventureSettle`
- `C2M_TrialChallenge` / `M2C_TrialResult`
- `C2M_RelicChooseNode` / `M2C_RelicSync`
- `C2M_EndlessStart` / `M2C_EndlessSettle`

## 6. 验收指标

- 14日内玩家至少完成2次大境跨越相关挑战。
- 天梯参与率与遗迹复玩率达到目标区间。
- PVE产出能覆盖主线关键养成需求。
