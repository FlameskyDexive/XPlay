# 06. 养成系统与构筑深度

## 1. 模块目标

- 让玩家通过“构筑选择”而非单一堆数值完成成长。
- 支撑不同境界阶段的流派分化与重构空间。
- 将养成结果直接反馈到渡劫与高难PVE挑战。

## 2. 核心养成子系统

## 2.1 功法系统

- 三体系：炼体、炼气、炼神。
- 支持双修流派，但引入冲突成本（槽位、资源、词条互斥）。
- 关键体验：不同功法组合决定战斗节奏与渡劫稳定性。

## 2.2 法宝系统

- 结构：主动技 + 被动词条。
- 关键机制：洗炼、继承、胚子培养。
- 目标：法宝选择成为“功能决策”，不只是战力加成。

## 2.3 灵兽系统

- 双定位：战斗辅助 + 资源增益。
- 主要能力：护主技能、增益光环、派遣加成。
- 要求：避免灵兽成为唯一强制答案，保留替代路线。

## 2.4 铭纹系统

- 中后期核心数值系统，承载流派差异。
- 通过套装词条与散件词条并存，支持“稳态流/爆发流/控制流”。
- 与遗迹、心魔幻境深度联动。

## 2.5 道心系统

- 主要影响雷劫抗性与心魔判定成功率。
- 防止战斗只看攻击，提供生存与风险管理维度。
- 道心成长建议绑定剧情抉择与阶段挑战。

## 3. 构筑策略框架

## 3.1 基础策略位

- 输出位：功法主系 + 法宝主动技。
- 生存位：道心抗性 + 减伤阵法 + 回复丹药。
- 控制位：铭纹词条 + 灵兽辅助技能。

## 3.2 流派示例

- 稳态渡劫流：高抗雷 + 高回复 + 中速输出。
- 速攻破劫流：高爆发 + 低容错，追求短时击破。
- 反制控制流：净化/免控/减CD，针对心魔机制。

## 4. 代码设计要点（ET）

## 4.1 建议包归属

- `cn.etetet.cultivation`：功法与道心。
- `cn.etetet.hero`：护道人/灵兽能力。
- `cn.etetet.item` / `cn.etetet.equipment`：法宝与铭纹承载。

## 4.2 建议组件

```csharp
public class GongfaBuildComponent : Entity, IAwake
{
    public int BodyGongfaId;
    public int QiGongfaId;
    public int MindGongfaId;
    public int ActiveDualPath;
}
```

```csharp
public class DaoHeartComponent : Entity, IAwake
{
    public int DaoHeartLevel;
    public int HeartDemonResist;
    public int TribulationResist;
}
```

## 4.3 建议系统

- `GongfaBuildSystem`：功法切换、冲突校验、属性重算。
- `ArtifactRefineSystem`：洗炼、继承与词条锁定。
- `InscriptionComposeSystem`：铭纹合成、套装激活、词条筛选。
- `DaoHeartGrowthSystem`：道心成长事件与雷劫抗性同步。

## 4.4 数值约束

- 单一子系统不应贡献超过总战力的40%。
- 控制“爆发流”上限，确保高难内容不被单套构筑跳过。
- 提供至少两条可行构筑路径以避免玩法收敛。

## 5. 配置与协议

## 5.1 配置表建议

- `GongfaConfig`
- `ArtifactConfig`
- `InscriptionConfig`
- `DaoHeartConfig`

## 5.2 协议建议

- `C2M_GongfaSet` / `M2C_GongfaSync`
- `C2M_ArtifactRefine` / `M2C_ArtifactResult`
- `C2M_DaoHeartUpgrade` / `M2C_DaoHeartSync`

## 6. 验收指标

- 不同境界段主流构筑多样性达到目标阈值。
- 渡劫成功率与道心成长有显著正相关。
- 铭纹/法宝系统使用率稳定提升。
