# 04. 放置修炼与洞府系统

## 1. 模块目标

- 构建“离线可成长、上线可决策”的修炼底盘。
- 保障碎片化体验下的稳定推进，不强迫长时间在线。
- 为突破与副本挑战持续提供修为与基础材料。

## 2. 玩法内容细化

## 2.1 打坐

- 基础修为来源，稳定、低门槛。
- 支持离线累计，上限默认24小时。
- 受功法、洞府、奇遇增益影响。

## 2.2 闭关

- 消耗资源换取高倍修为，适合冲击突破窗口。
- 闭关期间可触发随机事件，提供额外收益或轻惩罚。
- 支持“快速闭关券”缩短等待时间。

## 2.3 游历

- 派遣玩法，产出稀有材料与奇遇事件。
- 按区域设定风险等级与词条池。
- 支持“保底奇遇进度”，避免长期黑脸。

## 2.4 洞府

- 建筑化成长系统，提供常驻增益与自动产出。
- 建议建筑位：聚灵台、丹房、灵田、藏经阁、护阵台。
- 建筑升级应提供“即时提升 + 长线收益”双反馈。

## 3. 产出与效率设计

## 3.1 修为产出公式

```text
每小时修为 = 境界基础值 × 功法系数 × 洞府系数 × 奇遇加成 × (1 + 离线补偿)
```

## 3.2 友好性约束

- 离线收益不低于同阶段在线收益的70%。
- 每天首次登录触发“悟道时刻”短时加速。
- 自动修炼队列支持12小时计划，减少重复操作。

## 3.3 防通胀约束

- 高阶材料与低阶材料分池，避免单资源通吃。
- 洞府高等级产出增幅递减，防止后期数值失控。

## 4. 代码设计要点（ET）

## 4.1 建议包归属

- `cn.etetet.cultivation`：打坐/闭关/离线补算。
- `cn.etetet.adventure`：游历事件与奇遇结果。

## 4.2 建议组件

```csharp
public class OfflineCultivationComponent : Entity, IAwake
{
    public long LastCollectTime;
    public int OfflineCapHours;
    public long StoredCultivation;
    public long StoredMaterials;
}
```

```csharp
public class CaveComponent : Entity, IAwake
{
    public int CaveLevel;
    public Dictionary<int, int> BuildingLevelMap;
    public long LastProductTick;
}
```

## 4.3 建议系统

- `CultivationTickSystem`：定时累积修为与材料。
- `OfflineSettleSystem`：登录时补算离线收益并下发明细。
- `CaveBuildSystem`：建筑升级、产出结算、条件校验。
- `WanderDispatchSystem`：游历派遣、事件抽样、结果写回。

## 4.4 性能与安全

- 使用离散时间片结算，避免逐秒回放造成CPU抖动。
- 离线收益仅由服务端时间戳计算，防止本地改时作弊。
- 领取收益接口做幂等处理，避免重复发奖。

## 5. 配置与协议

## 5.1 配置表建议

- `CaveConfig`、`CaveBuildingConfig`。
- `CultivationRateConfig`、`OfflineCompensationConfig`。
- `WanderEventConfig`、`WanderRewardConfig`。

## 5.2 协议建议

- `C2M_CultivateStart` / `M2C_CultivateSync`
- `C2M_CaveCollect` / `M2C_CaveStatus`
- `C2M_WanderDispatch` / `M2C_WanderSettle`

## 6. 验收指标

- 日活用户中离线收益领取率达到预期阈值。
- 12小时队列使用率稳定增长。
- 闭关功能的突破转化率满足目标。
