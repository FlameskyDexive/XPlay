# 08. 经济与数值框架设计

## 1. 模块目标

- 建立“可长期增长但不过度通胀”的修仙经济框架。
- 确保主线关键资源由PVE稳定供给，SLG只提供效率增益。
- 通过曲线控制与反卡关机制降低挫败与流失。

## 2. 资源分层设计

## 2.1 核心成长资源（PVE稳定产出）

- 修为
- 灵气
- 渡劫材料
- 功法残卷

## 2.2 效率资源（可被SLG加成）

- 加速符
- 生产增益道具
- 宗门贡献

## 2.3 荣誉资源（展示导向）

- 称号点
- 赛季勋章

## 2.4 资源来源占比目标

| 资源类型 | 放置/PVE | 活动投放 | SLG | 说明 |
|---|---:|---:|---:|---|
| 修为 | >=80% | <=20% | <=5% | SLG仅通过加速或效率放大间接影响 |
| 渡劫材料 | >=95% | <=5% | 0% | 不允许SLG独占或主供给 |
| 功法残卷 | >=85% | <=15% | <=5% | SLG不可成为核心残卷主要来源 |
| 洞府成长材料 | >=80% | <=20% | <=10% | 可少量受宗门互助影响 |

## 3. 战力占比约束

- 修仙主系统：`70%-80%`
- PVE装备/法宝/铭纹：`15%-25%`
- SLG相关增益：`<=10%`

设计意图：玩家不参与SLG仍可保持主线强度竞争力。

- 以 `20-30分钟` 标准日常估算，SLG获得的效率收益不应超过同阶段主链 `3日` 产出的 `15%`。

## 4. 关键数值曲线

## 4.1 境界需求曲线

```text
小层需求修为 = A(境界) × 层数系数 × 成长倍率
大境界跃迁需求 = 当前大境9重需求 × 2.5~4.0
```

## 4.2 雷劫强度曲线

```text
雷劫强度 = 基础强度(境界) × 小层系数 × (1 - 失败补偿系数)
```

## 4.3 参数调优建议

- 小层系数线性+微指数混合，避免前期拖沓或后期爆炸。
- 大境跨越采用阶梯跃迁，强化里程碑感。
- 失败补偿系数上限与保底联动，控制连败体验。

## 5. 反卡关机制

## 5.1 纯修仙追赶约束

- 纯修仙玩家7日/30日进度应稳定达到轻度SLG玩家的 `85%-90%`。
- 若低于阈值，优先补主链资源供给与个人周目标，而不是继续提高SLG奖励。
- 任何活动投放导致该差距被拉大，都应触发资源来源审计。

## 5.2 基础反卡关手段

- 主线卡关超48小时触发“天机指引”任务链。
- 连续突破失败触发保底增益，避免长期硬卡。
- 关键养成资源均提供替代来源，防止单点堵塞。

## 6. 代码设计要点（ET）

## 6.1 建议包归属

- `cn.etetet.numeric`：通用数值计算与参数接口。
- `cn.etetet.cultivation`：境界/突破相关计算。
- `cn.etetet.adventure`：副本掉落与收益策略。

## 6.2 建议组件

```csharp
public class EconomyStatComponent : Entity, IAwake
{
    public long DailyCultivationGain;
    public long DailyCoreMaterialGain;
    public long DailyEfficiencyResourceGain;
    public int ContinuousBlockHours;
}
```

## 6.3 建议系统

- `RealmFormulaSystem`：境界需求与增益计算。
- `TribulationFormulaSystem`：雷劫强度与补偿计算。
- `EconomyMonitorSystem`：资源流入流出监控、越界告警。
- `AntiBlockSystem`：卡关识别、定向任务触发。
- `RewardSourceAuditSystem`：按资源来源标签审计纯PVE/活动/SLG占比。

## 6.4 数据治理

- 每日生成资源报表（按境界段分桶）。
- 每周校验SLG投放边界与主线资源占比。
- 版本前进行数值回放，避免活动导致经济穿透。

## 8. 验收指标

- 48小时以上卡关用户占比持续下降。
- 纯修仙玩家境界进度保持在目标区间（80%-90%）。
- 资源通胀率和回收率在安全阈值内。
